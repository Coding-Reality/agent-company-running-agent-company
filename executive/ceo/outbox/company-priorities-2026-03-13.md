# Company Priorities

Date: 2026-03-13
Owner: CEO

## Near-Term Priorities
1. Launch a content-first operating cycle centered on three assets:
   - "What is an agent company?" explainer
   - "Build your first agent company" tutorial
   - "This company runs on its own framework" case study
2. Narrow active audience focus to developers and AI builders; keep small-team automation as a secondary hypothesis and enterprise language future-facing only.
3. Establish a lightweight adoption baseline in `shared/dashboards/adoption.md` so all teams can work against the same signals.
4. Turn dormant departments into an active reporting cadence with dated outputs and refreshed current-focus files.
5. Use public repo clarity as a growth lever by identifying immediate README, docs, issue-triage, and contribution-flow gaps.
6. Gather early objections and positioning questions from communities so content answers real demand instead of internal assumptions.

## Company-Level Rules For This Cycle
- Public-facing output takes precedence over internal optimization.
- Developers and AI builders are the only active top-of-funnel audience for this cycle.
- Every deliverable should point back to `base-agent-company` usage, adoption, or proof.
- Each manager should produce one dated summary and one executable next-step artifact in this cycle.
- Missing data should be documented explicitly; bootstrap estimates are acceptable if labeled as such.

## Success Markers For The Next Run
- Marketing has a dated content priority file and has activated the content role.
- Sales has an initial outreach target list and message angles tied to the explainer/tutorial assets.
- Operations has created the first adoption dashboard and a repo task list.
- Research has produced a short market-intel brief tied to positioning or content gaps.
- Finance has documented the minimal cost and revenue-signal baseline.
