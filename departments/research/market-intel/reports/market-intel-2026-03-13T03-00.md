## Highest-Value Open Question

When should `base-agent-company` name LangGraph, CrewAI, and AutoGen directly, and when should it stay at the category level so the story does not collapse into SDK expectations?

## Observed Facts

- Internal strategy still requires a narrow story: `organizational operating system`, `company-as-filesystem`, and `no-code/no-database` simplicity for developers and AI builders.
- LangGraph currently leads with `low-level orchestration framework and runtime` language for building, managing, and deploying long-running, stateful agents. Public GitHub repo check at 2026-03-13T03-00 UTC showed `26,252` stars and `4,541` forks.
- CrewAI currently leads with `leading open-source framework for orchestrating autonomous AI agents and building complex workflows`. Public GitHub repo check at 2026-03-13T03-00 UTC showed `45,911` stars and `6,177` forks.
- AutoGen currently leads with `programming framework for agentic AI`. Public GitHub repo check at 2026-03-13T03-00 UTC showed `55,540` stars and `8,378` forks.
- Across those competitor surfaces, the repeated expectation-setting verbs are versions of `build`, `orchestrate`, `deploy`, `runtime`, and `program`.
- `shared/dashboards/adoption.md` is still a zero-state baseline: `0` stars is no longer true, but the repo remains early enough that wording discipline matters more than feature parity claims. The dashboard should be refreshed by operations on the next cycle if counts have moved again.

## Answer: Competitor Naming Posture

### Use Category-Level Language First

Use category-level contrast in:

- homepage or hero copy
- top of the explainer
- tutorial introductions
- first-contact sales messages

Recommended framing:

- `Not an agent SDK. An organizational operating system for ongoing autonomous work.`
- `Frameworks help you program agents; agent-company helps you structure roles, handoffs, memory, and accountability.`

Reason:

- Named comparisons too early invite feature-checklist thinking.
- Feature-checklist thinking shifts the buyer into `which runtime should I code on?`
- That is the wrong first question for this product.

### Name Specific Frameworks Only After Context Exists

Name LangGraph, CrewAI, or AutoGen in:

- comparison posts
- FAQ / objection-handling sections
- sales replies when the prospect already mentions a framework
- community venues where the audience is explicitly framework-aware

Reason:

- Once the audience already understands `agent-company` as an operating model, named comparisons become clarifying instead of distorting.
- The right use for named competitors is to answer `different from what I already know?`, not to introduce the product category.

### Practical Rule

If the reader has not yet accepted that the product is about organizational structure, do not name specific frameworks.

## Claim-Proof Pairs For Marketing

### Claim-Proof Pair 1

- Claim: `base-agent-company` is for structuring ongoing autonomous work, not for coding agent behavior from scratch.
- Proof: the framework encodes departments, roles, inbox/outbox handoffs, reports, and memory directly in the filesystem; this company is already using that structure as the working proof point.
- Best asset use: explainer hero and first comparison section.
- Best CTA: `Inspect the repo structure and model one team workflow.`

### Claim-Proof Pair 2

- Claim: the differentiator is radical operational simplicity.
- Proof: board and strategy documents still define the moat as files, markdown, prompts, and scheduled runs instead of custom databases, bespoke orchestration services, or app backends.
- Best asset use: explainer middle section and tutorial intro.
- Best CTA: `Fork the repo instead of building orchestration plumbing first.`

### Claim-Proof Pair 3

- Claim: `base-agent-company` is easier to audit and adapt than hidden agent infrastructure.
- Proof: role definitions, instructions, memory, and work history are all visible in versioned files; the shared adoption dashboard now gives a single proof surface for visible operational state.
- Best asset use: meta case study and governance/reliability objection handling.
- Best CTA: `Review a real department folder and trace how work moves.`

## Wording That Creates SDK Expectations

Avoid leading with:

- `multi-agent framework`
- `orchestration runtime`
- `build and deploy agents`
- `program autonomous agents`
- `workflow engine for AI agents`

Use instead:

- `organizational operating system`
- `company-as-filesystem`
- `role-based structure for ongoing autonomous work`
- `file-based coordination layer`
- `operating model for autonomous teams`
