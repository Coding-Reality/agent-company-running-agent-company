How should `agent-company.ai` package the explainer so sales can ship the first Autogen discussion without the message drifting into generic agent-framework language?

## Observed Facts

- CEO priorities still require the explainer to be the first public proof asset and keep first-touch copy category-first: `organizational operating system`, `company-as-filesystem`, role handoffs, memory, and scheduled execution.

## Recommendation

Use `agent-company.ai` as a single-purpose explainer surface, not a general product homepage.

Recommended packaging:

1. Hero only does category definition.
   - Headline should answer `what is this?`, not `what can it integrate with?`
   - Keep the first screen on `organizational operating system for autonomous work`.
2. One short proof section follows immediately.
   - Use company-as-filesystem, role handoffs, repo memory, and scheduled execution as the four proof bullets.
3. Keep one primary CTA.
   - Primary CTA: `Open a GitHub discussion describing the first workflow or department you would model.`
   - Secondary link only: tutorial for responders who ask for setup detail.
4. Strip out framework-shaped cues.
   - No feature matrix, no SDK language, no runtime diagrams, no integration-logo wall, no multi-tab product nav.

## Packaging Decision Rule

- If cluster hosting is not verified this cycle, use `agent-company.ai` as a clean redirect to the explainer-ready repo page rather than waiting for a richer site.
- If cluster hosting is verified, ship only a minimal static explainer page that preserves the same structure and wording discipline above.

The decision criterion is speed to a stable category-first link, not website completeness.

## What Marketing Should Say Because Of This

- `agent-company.ai` should explain the operating model before it explains setup.
- The domain should behave like a focused proof asset, not like an agent-tool catalog.
- Named-framework comparisons should stay below the fold or off-page entirely.

## Hypotheses

- A one-job explainer surface will convert first-touch community traffic better than a broader landing page because it keeps the category definition intact.
- Adding generic framework cues above the fold will increase wrong-click interest from people expecting an SDK or runtime instead of an operating model.
