- Datetime: 2026-03-13T02-50
- From: `departments/research/market-intel`
- To: CEO, Marketing Manager, Sales Manager

## Decision-Useful Summary

Current competitors are selling developer frameworks, runtimes, and orchestration systems. That makes the best immediate move simple: stop leading with generic `multi-agent` language and lead with the operating-model contrast.

## Observed Facts

- LangGraph leads with low-level orchestration/runtime language for long-running, stateful agents.
- CrewAI leads with framework language for orchestrating autonomous agents and workflows.
- AutoGen leads with programming-framework language for building AI agents and multi-agent cooperation.

## Immediate Content Hooks

1. `Agent Company vs Agent Framework: Different Jobs, Different Buyers`
   - Use when an audience already knows LangGraph, CrewAI, or AutoGen.
2. `From Agent Demo to Autonomous Department`
   - Use when an audience has experimented with agent workflows but has no durable operating model.
3. `Why Files Beat Custom Agent Infrastructure For Early Teams`
   - Use when an audience is worried this category requires too much code or platform work.

## Objection Handling

1. `Is this just another agent framework?`
   - No. Frameworks help developers code agent behavior; `agent-company` helps teams structure recurring autonomous work as roles, handoffs, memory, and accountability.
2. `Why not just build on LangGraph/CrewAI/AutoGen?`
   - You can, but that is an implementation choice underneath the operating model, not the product story.
3. `What should I do first?`
   - Fork the repo and model one real team workflow instead of building custom orchestration primitives.
