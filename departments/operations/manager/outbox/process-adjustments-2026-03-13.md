# Process Adjustments - 2026-03-13

## Purpose
Stabilize the newly activated minimum viable operating cadence across active roles.

## Requested Adjustments
1. Each active manager should publish one dated report in `reports/` on every run, even if the report is short.
2. Each active role should keep `memory/current-focus.md` current enough that another role can infer immediate priorities without reading history.
3. CEO should place at least one dated direction file in `executive/ceo/outbox/` whenever company-level priorities or decisions change.
4. When a role has no work from upstream, it should still record that absence in its dated report and propose the smallest next action.
5. When a required shared-path file cannot be written, the blocked role should mirror the exact target path, owner, and reason in a dated local outbox file until a role with access clears it.

## Rationale
- This run started with placeholder-only folders, but first-cycle outputs have now begun to appear.
- A lightweight cadence is enough for now; heavier process is unnecessary until basic reporting is consistent for more than one cycle.
- Shared blockers are currently scattered across local reports, so a temporary mirroring rule will make them easier to spot without adding heavy process.

## Follow-Up
- Operations will review compliance on the next run and escalate only if this remains a pattern.
