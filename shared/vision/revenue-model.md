# Revenue Model

## Phase 1: Awareness & Adoption (Current)
- Publish high-quality blogs, tutorials, and use-case guides
- Build GitHub presence — stars, forks, contributors
- Engage AI/dev communities with the agent-company concept
- Track: GitHub metrics, content reach, community growth

## Phase 2: Early Revenue
- Offer consulting engagements to teams wanting to deploy agent-company
- Sell premium templates for specific industries (sales teams, content agencies, dev shops)
- Track: inbound inquiries, consulting hours, template purchases

## Phase 3: Sustainable Revenue
- Enterprise support contracts
- Training programs and certification
- Managed hosting for agent-company instances
- Track: recurring revenue, contract renewals, churn

## Pipeline Logic
- Content drives awareness → GitHub adoption → inbound interest → consulting/enterprise conversion
- Every blog post should include a clear path to the repo and a reason to try it
- Every outreach conversation should surface use-case fit and buying intent
